📧 邮箱密码怎么获取

这个工具用一个发件邮箱把线索表格发出去。下面按你的邮箱类型选一种，拿到密码后填回控制台的「邮件设置」。注意：基本都不是用你平时登录的密码，而是单独生成的「应用专用密码」。

A. Gmail（@gmail.com）

  1. 开两步验证：打开 myaccount.google.com/security → 开启「两步验证 / 2-Step Verification」。
  2. 生成应用专用密码：打开 myaccount.google.com/apppasswords → 起个名字（如 projectintel）→ 生成 → 复制那 16 位密码。
  3. 回到控制台「邮件设置」：服务商选 Gmail，发件邮箱填你的 Gmail，密码粘刚才那 16 位 → 保存 → 发测试邮件。

B. 公司邮箱 @ravv.com（微软 Microsoft 365）

你们的 @ravv.com 邮箱托管在微软 Microsoft 365。比 Gmail 麻烦一点，因为微软默认会关掉「用密码直接发信」，可能需要 IT 管理员配合。

  1. （管理员做）开启 SMTP 发信权限：登录 admin.microsoft.com → 用户 → 选中该邮箱 → 「邮件 / Mail」标签 → 「管理电子邮件应用 / Manage email apps」→ 勾上 已通过身份验证的 SMTP（Authenticated SMTP） → 保存。
  2. 拿密码：
    • 若账号开了两步验证（MFA）：登录 mysignins.microsoft.com/security-info → 添加「应用密码 / App password」→ 复制。
    • 若没开两步验证：通常可直接用邮箱的登录密码。
  3. 回到控制台「邮件设置」：服务商选 Microsoft 365（会自动填 smtp.office365.com / 端口 587 / 不勾 SSL），发件邮箱填 你@ravv.com，密码粘上一步拿到的 → 保存 → 发测试邮件。
如果测试邮件报 “authentication failed / 5.7.x” 之类：多半是第 1 步没开，或公司整个租户禁用了基本认证。最快的替代办法——改用一个 Gmail 来发（按上面 A），@ravv.com 的人照样能收到，业务上没影响。

C. 其他邮箱（自定义）

找你邮箱服务商的「SMTP 设置」页，拿到：SMTP 服务器地址、端口（一般 465 或 587）、用户名（通常是完整邮箱）、密码（或应用专用密码）。在控制台「邮件设置」里服务商选「自定义」，按页面填即可。

小贴士：只有「发件邮箱」需要密码。收件人（拿线索表格的人）什么都不用弄，直接收邮件就行。
